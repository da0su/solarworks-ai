# Airtable ビュー設定ガイド（CEO向け）

Airtable APIではビュー作成ができないため、以下のビューをAirtable UI上で手動作成してください。
各テーブルの「+ ビューを作成」ボタンから「グリッド」を選択し、フィルタ・ソートを設定します。

---

## A. 販売管理

### ビュー1: 出品中
- フィルタ: 出品ステータス = 「出品中」
- ソート: 統一管理ID 昇順

### ビュー2: 売却済み
- フィルタ: 出品ステータス = 「売却済」
- ソート: 販売日 降順

### ビュー3: 未出品
- フィルタ: 出品ステータス = 「未出品」
- ソート: 統一管理ID 昇順

### ビュー4: 高利益順
- フィルタ: なし
- ソート: 即決時想定利益 降順

### 推奨列順（左から）
統一管理ID → 旧番号 → アイテム名 → 出品ステータス → 仕入合計 → 販売想定価格 → CEO決定価格 → 最低販売価格 → 即決価格 → 販売想定粗利 → CEO価格時想定粗利 → 実売価格 → 実利益 → タイトル → 説明文

---

## B. 利益分析

### ビュー1: S/A/Bランク
- フィルタ: 粗利ランク が S, A, B のいずれか
- ソート: 利益率 降順

### ビュー2: C以下
- フィルタ: 粗利ランク が C, D, E のいずれか
- ソート: 利益率 降順

### ビュー3: 利益率順
- フィルタ: なし
- ソート: 利益率 降順

### ビュー4: 想定利益順
- フィルタ: なし
- ソート: 想定利益 降順

### 推奨列順
統一管理ID → コインID → コイン名 → 仕入元 → 想定ヤフオク売価 → 合計コスト → 想定利益 → 利益率 → 粗利ランク → 投資優先度 → AIコメント

---

## C. ヤフオク相場履歴

### ビュー1: 直近順
- ソート: 落札日 降順

### ビュー2: 高値順
- ソート: 落札価格 降順

### ビュー3: PF70のみ
- フィルタ: グレード に「PF70」を含む
- ソート: 落札価格 降順

### 推奨列順
統一管理ID → コイン名 → 国 → 年号 → グレード → 鑑定会社 → 落札価格 → 落札日 → URL → 備考

---

## D. 在庫台帳

### ビュー1: 保有中
- フィルタ: 在庫ステータス = 「保有中」

### ビュー2: 出品中
- フィルタ: 在庫ステータス = 「出品中」

### ビュー3: 売却済
- フィルタ: 在庫ステータス = 「売却済」

### ビュー4: 棚卸対象
- フィルタ: 在庫ステータス が「保有中」または「出品中」
- ソート: 総原価 降順

### 推奨列順
統一管理ID → コインID → コイン名 → 在庫ステータス → 数量 → 取得単価 → 総原価 → 保管場所 → 日付 → メモ

---

## 不要テーブル削除

「Table 1」は空の初期テーブルです。
Airtable UI上で右クリック → 「テーブルを削除」で削除してください。
