# 楽天ROOM BOT - 運用ワークフロー

**バージョン**: v4.1
**更新日**: 2026-03-09

---

## 運用方針

- 楽天ROOM運用を**完全自動化**する
- CEOの承認なしでBOTが投稿・フォローを自動実行する
- CEOは**監視と例外対応のみ**
- 投稿前のCEO確認工程は不要

---

## BOT構造

ROOM BOTは**2つの独立したエンジン**で構成する。

| エンジン | 役割 | 日次目標 |
|----------|------|----------|
| **投稿BOT** | 商品投稿の自動実行 | 90〜100件/日（ランダム） |
| **フォローBOT** | ユーザーフォローの自動実行 | 150〜250件/日（ランダム） |

投稿BOTとフォローBOTは**別ワークフロー**として管理する。
日次の件数は固定せず、ランダムに変動させる（BOT対策）。

---

## 日次リセット

- **午前0時（00:00）** を1日のリセットポイントとする

```
━━━━━━ フォローBOT（完全自動） ━━━━━━

┌─────────────────────────────────────┐

               │

│ Step 2: セッション実行 [BOT自動]    │
│ ・1セッション = 最大50件            │
│ ・4セッションを1日の中で分散        │
│ ・セッション間に休憩を挟む           │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│ Step 3: フォロー実行 [BOT自動]      │
│ ・recommendUsersページを開く         │

│ ・既フォロー済み → スキップ          │
│ ・成功件数だけをカウント             │
│ ・target_countで必ず停止             │
│ ・間隔: ランダム化                   │
│ ・一定件数ごとにランダム休憩         │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│ Step 4: ログ記録 [BOT自動]          │
│ ・FOLLOW_LOG.json に記録             │
│ ・target_count / success_count       │
│ ・skip_count / error_count           │
└─────────────────────────────────────┘
```

---

## 件数制御ルール（投稿・フォロー共通）

1. `target_count` で指定した件数で**必ず停止**する
2. 既処理済み（既投稿・既フォロー）はカウントしない
3. 実際に成功した件数（`success_count`）だけをカウントする
4. 上限に達したら即停止する
5. ログに以下を必ず保存する:
   - `target_count` — 目標件数
   - `success_count` — 成功件数
   - `skip_count` — スキップ件数
   - `error_count` — エラー件数

## CEOの役割

| 役割 | 内容 |
|------|------|
| 監視 | 日次ログの確認 |
| 例外対応 | エラー・警告時の判断 |
| 設定変更 | 件数・間隔・比率の調整 |
| ログイン | 初回ログイン（パスワード入力） |

投稿前確認は**不要**。

## ファイルの流れ

```
投稿BOT:
  リサーチ結果   → 05_CONTENT/rakuten_room/drafts/YYYY-MM-DD_research.md
  投稿文         → 05_CONTENT/rakuten_room/posts/YYYY-MM-DD_post_final.md
  投稿ログ       → 05_CONTENT/rakuten_room/history/POST_LOG.json
  日次ログ       → 05_CONTENT/rakuten_room/history/daily/YYYY-MM-DD_daily_log.json

フォローBOT:
  フォローログ   → 05_CONTENT/rakuten_room/history/FOLLOW_LOG.json
  日次ログ       → 05_CONTENT/rakuten_room/history/daily/YYYY-MM-DD_daily_log.json

共通:
  エラーログ     → 08_AUTOMATION/room_bot_v2/data/logs/error.log
  スクリーンショット → 08_AUTOMATION/room_bot_v2/data/screenshots/
```
