## 運用方針

- CEOの承認なしでBOTが自動実行する
- CEOは**監視と例外対応のみ**
- 投稿前確認は不要

## 日次リセットとスケジュール

- **午前0時（00:00）** を1日のリセットポイントとする

### 投稿スケジュール

| バッチ | 開始時刻 | 件数 |
|--------|----------|------|
| バッチ1（夜） | **0:00〜0:30**（ランダム開始） | 50件 |
| バッチ2（朝） | **7:00〜12:00**（ランダム開始） | 残り（40〜50件） |

開始時刻を固定にせず、範囲内でランダムに決定する（BOT対策）。

### フォロースケジュール

| セッション | 件数 | 時間帯 |
|-----------|------|--------|
| セッション1 | 最大50件 | いつでもOK |
| セッション2 | 最大50件 | いつでもOK |
| セッション3 | 最大50件 | いつでもOK |
| セッション4 | 最大50件 | いつでもOK |

- 1セッション = **最大50件**
- **4セッション**に分散して実行する
- セッション間にランダム休憩を挟む
- 一定件数（10〜15件）ごとにランダム休憩を入れる
- 短時間に集中しすぎないようにする

## 投稿BOTルール

| ルール | 内容 |
|--------|------|
| 日次件数 | 90〜100件（ランダム） |
| 一括投稿 | **禁止** |
| バッチ分割 | 50件 + 残り |
| バッチ1開始 | 0:00〜0:30（ランダム） |
| バッチ2開始 | 7:00〜12:00（ランダム） |
| 投稿間隔 | ランダム化 |
| 導線A | direct（商品ページ→ROOM投稿）70% |
| 導線B | influencer（派生投稿）30% |
| 導線選択 | ランダム（比率は設定で変更可能） |

## フォローBOTルール

| ルール | 内容 |
|--------|------|
| 日次件数 | 150〜250件（ランダム） |
| セッション分割 | **50件 × 4セッション** |
| 起点ページ | recommendUsers |
| フォロー順序 | 上から順 |
| 既フォロー | スキップ（カウントしない） |
| カウント対象 | 新規フォロー成功数のみ |
| 停止条件 | target_countに達したら必ず停止 |
| 間隔 | ランダム化 |
| セッション内休憩 | 10〜15件ごとにランダム休憩 |
| セッション間休憩 | ランダム休憩 |

## 件数制御（投稿・フォロー共通）

```
post_target_count = 90〜100（日次ランダム）
```

## エラー・制限時の停止ルール

### 即時停止（赤信号）

| 症状 | 対応 |
|------|------|
| アカウント一時停止・凍結 | 全停止。24時間以上待機 |
| 「操作が多すぎます」警告 | 全停止。12時間待機 |
| 投稿が非表示になる | 投稿停止。原因調査 |
| ログインできなくなる | 全停止。CEO判断を待つ |

### 減速（黄信号）

| 症状 | 対応 |
|------|------|
| フォロー時にエラーが出る | フォロー数を半分に |
| 投稿速度が遅くなる | 投稿間隔を2倍に |
| フォロワーが急に減る | フォロー停止。2日間観察 |

### 回復手順
1. 停止してから**最低12時間**待つ
2. Test Aの数値に戻してリスタート
3. 3日間安定したら元のPhaseに復帰

### 絶対にやらないこと
- 警告が出ているのに操作を続ける
- 制限解除直後にフル稼働する
