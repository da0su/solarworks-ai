# 楽天ROOM 投稿BOT マニュアル

**バージョン**: v4.0（最終運用仕様）
**更新日**: 2026-03-09

---

## 運用方針

- BOTが投稿を**完全自動実行**する。CEO承認は不要
- CEOは監視と例外対応のみ

---

## 1. 日次件数

| 項目 | 値 |
|------|-----|
| 日次投稿数 | **90〜100件**（ランダム） |
| バッチ分割 | **50件 × 2回** |
| バッチ1開始 | **0:00**（午前0時） |
| バッチ2開始 | 12:00 |
| 日次リセット | 午前0時 |

毎日同じ件数にしない（BOT対策）。日次の件数は90〜100の間でランダムに決定する。
バッチ1は50件固定、バッチ2で残り（40〜50件）を消化する。

---

## 2. 投稿導線

### 導線A: direct（商品ページから直接投稿）

1. 楽天ROOM内検索で商品を探す
2. 「コレ!」ボタンをクリック
3. レビュー文を入力
4. 「完了」をクリックして投稿

### 導線B: influencer（インフルエンサー投稿から派生）

1. 同ジャンルのインフルエンサーの投稿を閲覧
2. 商品に「コレ!」
3. 自分のレビュー文に差し替えて投稿

### 導線の比率設定

```
direct_post = 70%
influencer_route = 30%
```

A/B導線は**ランダムで選択**する。比率は設定で変更可能。

### 技術メモ

- 「コレ!」→ 完了ボタン: `button.button-red.collect-btn`
- 座標クリックでは反応しない → JSクリック必須
- `document.querySelector('button.button-red.collect-btn').click()`

## 3. 件数制御ルール（必須）

```
post_target_count = 90〜100（日次ランダム）
```

### 基本ルール
1. `target_count` で指定した件数で**必ず停止**する
2. 成功した投稿だけをカウントする
3. 上限に達したら**即停止**する
4. ログに以下を保存する:
   - `target_count` — 目標件数
   - `success_count` — 成功件数
   - `skip_count` — スキップ件数
   - `error_count` — エラー件数

### カウントロジック
```
for each product:
    if success_count >= target_count:
        STOP  ← ループの最初で必ず確認
    try post:
        if success:
            success_count++
        else:
            error_count++
```

### 間隔ルール
- **ランダム化**: 固定間隔にしない
- 同じジャンルの連続投稿は**最大3件**まで

## 5. エラーハンドリング

| エラー | 対処 |
|--------|------|
| 商品が見つからない | スキップして次へ |
| 「コレ!」ボタンがない | スキップして次へ |
| 投稿が反映されない | 5秒待ってリトライ（1回のみ） |
| 「操作が多すぎます」警告 | **即停止**。CEOに報告 |
| セッション切れ | **即停止**。CEOに報告 |

---

## 6. ログイン

- ログインはCEO手動操作（変更不可）
- BOTはセッション有効中のみ動作する
- セッション切れを検知したら即停止
