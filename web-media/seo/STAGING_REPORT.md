## ▶ 起動後に何が起こるか

```
毎分ポーリング:
  ├── キュー確認 → 6件未満なら記事自動生成（Claude API）
  ├── 時刻確認 → 08/11/14/17/20/23時に1件 draft 投稿

WP管理画面での作業（手動）:
  draft記事を確認 → 問題なければ publish に変更
```

## ▶ publish移行条件（テックさん明文化）

以下の**全4条件**が揃うまで publish は禁止:

| # | 条件 | 確認方法 |
|---|-----|---------|
| 1 | A8リンク差し替え完了 | `.env` の `TABINAYI_LINK` がプレースホルダーでないこと |
| 2 | 定刻投稿1回確認 | daemon起動後、定刻にdraft記事がWPに追加されていること |
| 3 | draft記事の品質確認 | WP管理画面で記事本文・CTA・リンクを目視確認 |
| 4 | カテゴリ・CTA最終確認 | WP管理画面でカテゴリが正しく、CTAが3箇所あること |

**条件達成後の操作:**
```
WP管理画面 → 投稿一覧 → draft記事を選択 → ステータスを「公開」に変更
```
または `.env` の `status="draft"` → `status="publish"` に変更してdaemon再起動。
